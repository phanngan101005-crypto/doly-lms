Luồng 1: Xác thực & Đăng nhập

3 bảng · 4 vai trò
▼
Hệ thống hỗ trợ 4 vai trò: Admin, Student, Teacher, Parent. Xác thực qua email + password (bcrypt), hỗ trợ Google Sign-In và EmailJS OTP cho đăng ký. Phiên làm việc quản lý bằng auth_token.

Quy trình Đăng ký

1
Người dùng nhập thông tin đăng ký
Họ tên, email, mật khẩu, chọn vai trò (student/teacher/parent)
2
Gọi API gửi OTP qua EmailJS
Frontend gọi EmailJS SDK với service_id, template_id, public_key
→ EmailJS (bên thứ ba) → Hộp thư người dùng
3
Lưu mã OTP vào database
Tạo bản ghi trong user_verifications với thời hạn 5 phút
→ INSERT INTO user_verifications (email, otp_code, expires_at)
4
Người dùng nhập mã OTP → Xác thực
Kiểm tra OTP còn hạn, đánh dấu is_verified = TRUE
→ UPDATE user_verifications SET is_verified = TRUE WHERE email = ?
5
Tạo tài khoản users
Mật khẩu được băm bằng bcrypt trước khi lưu
→ INSERT INTO users (full_name, email, password_hash, role, status) VALUES (?, ?, bcrypt(?), ?, 'active')

Quy trình Đăng nhập

1
Gửi email + password + role lên API
POST /doly/api/auth_login.php — JSON body
2
Backend kiểm tra thông tin
Tìm user theo email, kiểm tra status='active' + role khớp, so sánh bcrypt
→ SELECT * FROM users WHERE email = ? AND role = ?
3
Tạo session token
Sinh auth_token ngẫu nhiên (bin2hex(random_bytes(64))) và lưu vào user_sessions
→ INSERT INTO user_sessions (user_id, auth_token, expires_at) VALUES (?, ?, ?)
4
Trả về token + thông tin user
Frontend lưu vào localStorage/sessionStorage, chuyển hướng theo role
→ Response JSON: { user_id, full_name, email, role, auth_token, expires_at }

Bảng liên quan

users (PK) user_verifications user_sessions (FK→users)

Luồng 2: Quản lý Lớp học

2 bảng · 2 vai trò
▼

Quy trình tạo & tham gia lớp

1
Giáo viên tạo lớp học
Nhập tên lớp, môn, cấp lớp. Hệ thống tự sinh class_code (mã mời ngẫu nhiên)
→ INSERT INTO classes (name, subject, grade_level, teacher_id, class_code) VALUES (?, ?, ?, ?, ?)
2
Giáo viên chia sẻ mã mời cho học sinh
class_code được hiển thị trong giao diện lớp học, có thể copy
3
Học sinh nhập mã mời → tham gia lớp
Kiểm tra class_code tồn tại, thêm bản ghi vào class_members
→ SELECT * FROM classes WHERE class_code = ? AND status = 'active'
→ INSERT INTO class_members (class_id, student_id) VALUES (?, ?)
4
Xem danh sách lớp + quản lý
Giáo viên xem/xoá học sinh. Học sinh xem danh sách lớp của mình
→ SELECT u.* FROM class_members cm JOIN users u ON cm.student_id = u.id WHERE cm.class_id = ?

Bảng liên quan

classes (PK) class_members (FK→classes, FK→users) users (FK→teacher_id)

Luồng 4: Câu hỏi & Kiểm tra

6 bảng · 4 loại câu hỏi
▼
Hệ thống hỗ trợ 4 loại câu hỏi: Trắc nghiệm (multiple_choice), Đúng/Sai (true_false), Điền khuyết (fill_blank), Ghép cặp (matching). Mỗi loại lưu ở bảng con riêng.

Quy trình tạo đề kiểm tra

1
Giáo viên tạo câu hỏi trong ngân hàng
Chọn loại câu hỏi, nhập nội dung, đáp án, độ khó, điểm XP
→ INSERT INTO question_bank (subject_id, lesson_id, question_text, question_type, difficulty, ...)
→ INSERT INTO question_options (question_id, option_text, is_correct, order_index) [nếu là trắc nghiệm]
→ INSERT INTO fill_blank_answers (question_id, correct_answer, position) [nếu là điền khuyết]
→ INSERT INTO matching_pairs (question_id, left_text, right_text) [nếu là ghép cặp]
2
Giáo viên tạo đề thi từ ngân hàng
Tạo exams, thêm câu hỏi vào exam_questions với điểm số từng câu
→ INSERT INTO exams (title, subject_id, created_by, time_limit, max_score, type) VALUES (?, ?, ?, ?, ?, ?)
→ INSERT INTO exam_questions (exam_id, question_id, order_index, score) VALUES (?, ?, ?, ?)
3
Học sinh làm bài kiểm tra
Hiển thị câu hỏi theo order_index, theo dõi thời gian (time_limit)
→ SELECT eq.*, qb.* FROM exam_questions eq JOIN question_bank qb ON eq.question_id = qb.id WHERE eq.exam_id = ? ORDER BY eq.order_index
4
Nộp bài → tự động chấm điểm
So sánh đáp án, tính điểm, cập nhật pass_count cho exams
→ UPDATE exams SET pass_count = pass_count + 1 WHERE id = ?

Bảng liên quan

question_bank (PK) question_options (FK→question_bank) fill_blank_answers (FK→question_bank) matching_pairs (FK→question_bank) exams (PK) exam_questions (FK→exams, FK→question_bank)

Luồng 5: Giao bài & Chấm điểm

4 bảng · 3 vai trò
▼

Quy trình giao bài → làm bài → chấm điểm

1
Giáo viên giao bài tập cho lớp
Chọn lớp, chọn đề thi (hoặc tạo mới), đặt hạn nộp. Trạng thái 'draft' hoặc 'published'
→ INSERT INTO assignments (class_id, exam_id, title, assigned_by, due_date, status) VALUES (?, ?, ?, ?, ?, 'published')
2
Học sinh nhận bài và làm bài
Xem danh sách bài tập cần làm, bấm "Làm bài" → vào giao diện làm bài
→ SELECT a.*, e.* FROM assignments a JOIN exams e ON a.exam_id = e.id WHERE a.class_id IN (SELECT class_id FROM class_members WHERE student_id = ?)
3
Học sinh nộp bài
Tạo bản ghi student_assignments, lưu từng câu trả lời vào student_answers. Nếu quá hạn → status='late'
→ INSERT INTO student_assignments (assignment_id, student_id, status, submitted_at)
→ INSERT INTO student_answers (student_assignment_id, question_id, answer_text, selected_option_id, ...)
4
Chấm điểm (tự động + thủ công)
Câu trắc nghiệm tự động chấm. Giáo viên vào bảng điểm, xem bài, nhập điểm tay cho câu tự luận
→ UPDATE student_answers SET is_correct = ?, score_earned = ? WHERE id = ?
→ UPDATE student_assignments SET score = ?, status = 'graded', graded_by = ?, graded_at = NOW() WHERE id = ?
→ INSERT INTO grades (student_id, class_id, subject_id, assignment_id, score, max_score)
5
Xem bảng điểm tổng hợp
Giáo viên xem điểm tất cả học sinh. Học sinh xem điểm cá nhân. Phụ huynh theo dõi qua link
→ SELECT g.*, a.title, u.full_name FROM grades g JOIN assignments a ON g.assignment_id = a.id JOIN users u ON g.student_id = u.id WHERE g.class_id = ? ORDER BY g.student_id

Bảng liên quan

assignments (PK) student_assignments (FK→assignments, FK→users) student_answers (FK→student_assignments, FK→question_bank) grades (FK→users, FK→classes, FK→subjects)

Luồng 6: Diễn đàn

4 bảng · 4 chủ đề
▼

Quy trình đặt câu hỏi & tương tác

1
Tạo bài viết mới
Chọn chủ đề (topic), nhập tiêu đề + nội dung. Admin duyệt nếu status = 'pending'
→ INSERT INTO forum_posts (user_id, title, content, topic_id, status) VALUES (?, ?, ?, ?, 'approved')
2
Người dùng khác bình luận
Viết câu trả lời, có thể đánh dấu is_best_answer
→ INSERT INTO forum_comments (post_id, user_id, content) VALUES (?, ?, ?)
3
Vote bài viết
Upvote hoặc downvote. Mỗi user chỉ vote 1 lần (UNIQUE constraint)
→ INSERT INTO forum_votes (post_id, user_id, vote_type) VALUES (?, ?, 'up') ON DUPLICATE KEY UPDATE vote_type = ?
4
Đánh dấu đã giải quyết
Chủ bài viết hoặc admin đánh dấu is_resolved = TRUE, chọn câu trả lời đúng nhất
→ UPDATE forum_posts SET is_resolved = TRUE WHERE id = ?
→ UPDATE forum_comments SET is_best_answer = TRUE WHERE id = ?
Kiểm duyệt: Admin có thể duyệt bài (pending → approved), gắn cờ (flagged), hoặc xoá (deleted). Thống kê: Mỗi lần xem bài viết tăng view_count.

Bảng liên quan

forum_topics (PK) forum_posts (FK→users, FK→forum_topics) forum_comments (FK→forum_posts, FK→users) forum_votes (FK→forum_posts, FK→users)

Luồng 7: Gamification

6 bảng · XP + Streak + Achievements
▼

Quy trình kiếm XP và mở huy hiệu

1
Học sinh thực hiện hành động kiếm XP
Làm quiz (+XP), nộp bài tập (+XP), điểm danh hàng ngày (+XP), đặt câu hỏi diễn đàn (+XP)
2
Ghi nhận XP và lịch sử giao dịch
Cập nhật total_xp trong user_xp. Ghi log vào xp_transactions
→ INSERT INTO xp_transactions (user_id, amount, source, reference_id) VALUES (?, +?, ?, ?)
→ UPDATE user_xp SET total_xp = total_xp + ? WHERE user_id = ?
3
Cập nhật streak và điểm danh
Kiểm tra lần hoạt động cuối, nếu liên tiếp → tăng streak
→ INSERT INTO daily_attendance (user_id, attendance_date, xp_earned) VALUES (?, CURDATE(), ?) ON DUPLICATE KEY ...
→ UPDATE user_streaks SET current_streak = IF(last_active_date = CURDATE() - 1, current_streak + 1, 1), last_active_date = CURDATE() WHERE user_id = ?
4
Kiểm tra và trao huy hiệu
Sau mỗi hành động, kiểm tra xem user có đủ điều kiện mở huy hiệu mới không
→ SELECT * FROM achievements WHERE id NOT IN (SELECT achievement_id FROM user_achievements WHERE user_id = ?)
→ [Nếu đủ điều kiện] INSERT INTO user_achievements (user_id, achievement_id) VALUES (?, ?)
5
Cập nhật rank
Tính rank dựa trên total_xp: Đồng, Bạc, Vàng, Kim Cương
→ UPDATE user_xp SET current_rank = CASE WHEN total_xp >= 1000 THEN 'Kim Cuong' WHEN total_xp >= 500 THEN 'Vang' WHEN total_xp >= 200 THEN 'Bac' ELSE 'Dong' END WHERE user_id = ?

Bảng liên quan

user_xp (FK→users, UNIQUE) xp_transactions (FK→users) user_streaks (FK→users, UNIQUE) daily_attendance (FK→users) achievements (PK) user_achievements (FK→users, FK→achievements)

Luồng 8: Phụ huynh

1 bảng trung gian
▼

Quy trình liên kết phụ huynh - học sinh

1
Học sinh tạo mã liên kết
Trong trang cài đặt cá nhân, học sinh tạo link_code để gửi cho phụ huynh
→ INSERT INTO parent_student_links (parent_id, student_id, link_code, status) VALUES (NULL, ?, ?, 'pending')
2
Phụ huynh nhập mã liên kết
Đăng nhập tài khoản parent → nhập link_code → xác nhận liên kết
→ SELECT * FROM parent_student_links WHERE link_code = ? AND status = 'pending'
→ UPDATE parent_student_links SET parent_id = ?, status = 'active' WHERE link_code = ?
3
Phụ huynh theo dõi tiến độ
Xem điểm số, tiến trình học, lịch sử hoạt động của con qua dashboard phụ huynh
→ SELECT g.*, a.title, u.full_name FROM grades g JOIN assignments a ON ... JOIN users u ... WHERE g.student_id IN (SELECT student_id FROM parent_student_links WHERE parent_id = ? AND status = 'active')

Bảng liên quan

parent_student_links (FK→users parent, FK→users student) grades (đọc) user_xp (đọc) user_streaks (đọc)

Người dùng đọc thông báo

Đăng bài tin tức (news), sửa FAQ. Tin tức có thể ghim (is_pinned) và xếp lịch xuất bản

Luồng 10: AI Chat (Doly Tutor)

1 bảng · Phương pháp Socratic
▼
DOLY AI sử dụng phương pháp Socratic: không giải bài tập trực tiếp cho học sinh, mà đặt câu hỏi gợi mở để học sinh tự suy luận. Mỗi tin nhắn đều lưu vào cơ sở dữ liệu để duy trì ngữ cảnh hội thoại.

Quy trình trò chuyện với AI

1
Học sinh gửi câu hỏi
Nhập câu hỏi vào chat box. Hệ thống gửi kèm context (bài học hiện tại, nếu có)
2
Lưu tin nhắn của user vào database
→ INSERT INTO ai_chat_history (user_id, role, message, context) VALUES (?, 'user', ?, ?)
3
Lấy lịch sử chat để xây dựng prompt
Lấy N tin nhắn gần nhất làm ngữ cảnh cho AI
→ SELECT * FROM ai_chat_history WHERE user_id = ? ORDER BY created_at DESC LIMIT 20
4
AI xử lý và trả lời
API AI nhận prompt (bao gồm lịch sử + câu hỏi mới), sinh câu trả lời theo phong cách Socratic
5
Lưu câu trả lời của AI
Lưu vào database để duy trì hội thoại xuyên suốt
→ INSERT INTO ai_chat_history (user_id, role, message, context) VALUES (?, 'assistant', ?, ?)

Bảng liên quan

ai_chat_history (FK→users)
